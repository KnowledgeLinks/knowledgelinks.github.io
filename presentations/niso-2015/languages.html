Languages

Case Study: Looking at the "English Language"

How is "English" used in catalog systems?

  • as a Subject/Topic: A book written about the English Language
  • the Language of production: A book written in english
  • A description of the catalog system language: A comment written in english
  • Primary language of an author
  • Language of a region or country
  • As a classifier: Library of Congress Classification

In a classical implementation of Relational database "English" would most likely be listed in multiple tables associated with the appropriate usage

Moving English to RDF

No more tables!!! One ID to rule them all

The ID becomes your URI: The "Unique Resource Identifier" for the concept of the "English Language"

But wait a minute??? I have five different Identifiers for English in my old system for English. No Worries…just add an alias (owl:sameAs tag)

How to describe the Usage

Telling the system how to use a resource can be done by adding a RDF Type tag to your resource

Connecting "English" to outside resources

Just add another owl:sameAs tag: 

<http://myNameSpace.org/english>
	owl:sameAs	<http://dbpedia.org/resources/English_Language>,
